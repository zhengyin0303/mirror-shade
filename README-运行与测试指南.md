# 镜色 MirrorShade · 原型运行与测试指南

原型为单文件 `mirror-shade.html`,无需安装任何东西,人脸识别模型(MediaPipe)由浏览器从 CDN 自动加载,**所有画面只在设备本地处理,不上传**。

## 一、怎么跑起来

浏览器规定:调用摄像头必须通过 `https://` 或 `localhost` 访问,直接双击打开本地文件大概率不行。三种方式按简单程度排序:

**方式 A · 手机实测(推荐,最贴近真实体验)**
把 `mirror-shade.html` 拖进任意免费静态托管(如 Netlify Drop:浏览器打开 app.netlify.com/drop,把文件拖进去,10 秒得到一个 https 链接),用手机浏览器打开该链接即可。Vercel、GitHub Pages 同理。

**方式 B · 电脑快速预览**
电脑装有 Python 的话,在文件所在目录运行 `python3 -m http.server 8000`,然后 Chrome 打开 `http://localhost:8000/mirror-shade.html`,用电脑前置摄像头测试。

**方式 C · 电脑起服务 + 手机访问**
方式 B 起服务后,手机访问 `http://电脑IP:8000/...` 会被浏览器以"非安全来源"拒绝摄像头,需配合内网穿透工具(如 `npx localtunnel --port 8000`)获得 https 地址。嫌麻烦直接用方式 A。

首次打开需加载约 3–10 秒模型(取决于网速),之后浏览器有缓存会快很多。

## 二、功能地图

| 区域 | 操作 |
|------|------|
| 启动页 | 点"开启镜子"→授权摄像头→自动进入肤色分析(约2秒) |
| 肤色结果条 | 顶部显示"你偏暖调/冷调…",分析完成自动涂上第一支推荐色;点"不太准?"可手动改 |
| 色板 | 横向滑动,带★的是为你的肤色推荐的显白色;点击即换色,界面强调色会跟着口红色变 |
| 一键试色 | 默认模式,点色秒上唇 |
| 亲手上妆 | 用手指在屏幕的嘴唇上来回涂抹,薄涂叠加越涂越浓;涂出界自动裁掉;可切大刷/小刷/卸妆棉(局部擦除) |
| 唇型 | 出现虚线唇线和8个白色控制点,拖动调整;预设:原生/微笑唇/嘟嘟唇/M唇;"对称"开关默认开;"完成"返回 |
| 浓度滑杆 | 薄涂 ↔ 厚涂 |
| 哑光/水光 | 质地切换,水光带下唇高光 |
| 按住对比 | 按住显示素颜,松手恢复妆容 |
| Clear | 一键卸妆 |
| 右上圆钮 | 拍照保存当前画面 |
| 缩放 | 画面右上角"1x"按钮点击循环 1x/1.5x/2x;也可双指捏合(1x–3x)或电脑滚轮;变焦中心自动跟随面部 |

## 三、实测清单(对照项目方案第七节)

逐项打勾,发现问题记下现象和当时的环境(机型/光线/动作):

- [ ] **张嘴护牙**:微笑、大笑、说"诶——"露出牙齿,颜色不染牙齿、不进口腔
- [ ] **运动跟随**:左右转头、点头、说话,妆容贴合无明显漂移抖动
- [ ] **涂抹锁区**:手指故意涂到嘴唇外,颜色不出界
- [ ] **叠涂曲线**:同一处涂 1 遍 / 5 遍 / 10 遍,浓度递增是否自然
- [ ] **涂抹跟随**:涂一半→转头→涂好的部分应跟着嘴走不错位
- [ ] **唇型编辑**:拖嘴角做微笑唇,上色后扩出的唇型是否自然
- [ ] **肤色分析**:在自然光、暖黄灯、冷白灯下各测一次,记录冷暖判定结果
- [ ] **多人样本**:请不同肤色、不同唇色(深唇/浅唇)的朋友各测一轮,重点看深唇显色
- [ ] **质地**:哑光与水光的差异是否能一眼看出,水光高光位置是否合理
- [ ] **性能**:画面是否流畅(目标≥20fps 观感),手机是否发烫

## 四、已知边界(原型阶段如实声明)

1. 渲染参数(混合比例、羽化、高光强度)是首版估值,需按真人实测调优——这正是原型的目的;
2. 肤色冷暖判定受环境光影响,暖黄灯下偏差大,已设手动修正兜底;
3. 唇色基底自适应(深唇显色补偿)尚未实现,列入下一迭代;
4. 旧版 iOS(Safari 18 以前)蒙版边缘羽化会退化为硬边,功能不受影响;
5. 仅支持单人脸。
