# 镜色 MirrorShade · v3.0 UI 品类化重构规格

> 作者:PM | 日期:2026-06-12 | 实现方:Code | 基线:v2.4(tag)
> 性质:信息架构重构 + 品类能力补全。渲染算法不动,变的是组织方式与作用域。

---

## 0. 需求背景(一句话)

v2.x 在扁平页签里混排了"操作模式"(试色/上妆/唇型)和"品类"(腮红/眉妆/眼影),真机使用混乱。重构为两级结构:**一级选品类,二级选模式**;Clear、按住对比、质地、浓度全部收窄为**当前品类作用域**。

## 1. 信息架构

### 1.1 一级页签(品类,固定 5 个,不滑动)

```
唇妆 | 腮红 | 眉妆 | 眼影 | 衣橱          (拍照快门钮保持在行尾)
```

### 1.2 二级模式(品类内,chips 行)

| 品类 | 二级模式 | 模式内工具 |
|------|---------|-----------|
| 唇妆 | 一键试色 / 亲手上妆 / 唇型 | 上妆:大刷/小刷/卸妆棉;唇型:原生/微笑唇/嘟嘟唇/M唇/对称/完成 |
| 腮红 | 一键上妆 / 亲手上妆 | 一键:打法 苹果肌/斜扫显瘦/晒伤腮红;亲手:大刷/小刷/卸妆棉 |
| 眉妆 | (无二级,直接工具)| 眉型:原生/加粗/挑眉 |
| 眼影 | 一键上妆 / 亲手上妆 | 一键:单色/外眼角渐变;亲手:大刷/小刷/卸妆棉 |
| 衣橱 | (维持 v2.4)| ＋自定义/导出/导入;点选口红 = 应用到唇妆品类 |

### 1.3 面板行结构(自上而下)

1. 一级页签行(含拍照钮)
2. 色板行(横滑,当前品类的色板;衣橱页签 = 衣橱网格)
3. 二级模式 + 模式工具行(横滑;眉妆只有工具)
4. 底部行:浓度滑杆 + 质地 seg(按品类显隐)+ 按住对比 + Clear

## 2. 品类能力矩阵(权威定义)

| 能力 | 唇妆 | 腮红 | 眉妆 | 眼影 |
|------|------|------|------|------|
| 一键上妆 | ✅ | ✅(三打法)| ✅(即选色)| ✅(单色/渐变)|
| 亲手涂抹 | ✅(已有)| ✅ **新增** | ❌(见 §2.1)| ✅ **新增** |
| 形状编辑 | ✅ 唇型 | — | ✅ 眉型(已有)| — |
| 质地 | 哑光/水光(已有)| **哑光/微闪** 新增 | **无,隐藏质地行** | **哑光/珠光** 新增 |
| 浓度 | ✅ 独立记忆(已有)| ✅ | ✅ | ✅ |
| 按住对比 | 仅卸当前品类 | 同 | 同 | 同 |
| Clear | 仅清当前品类 | 同 | 同 | 同 |

### 2.1 产品决策记录(三处"自己发挥"的裁定)

- **腮红质地 = 哑光/微闪**:真实腮红的两大形态;微闪渲染 = 蒙版内叠加低强度 screen 提亮(复用唇部水光管线,强度减半、范围更散)。
- **眼影质地 = 哑光/珠光**:珠光 = 同上 screen 提亮,强度介于唇水光与腮红微闪之间。
- **眉妆无质地、无手涂**:现实眉妆没有光泽维度;手指描眉在手机上必然画歪(眉是毫米级精细描画,非晕染动作),做了是负体验。眉妆 = 选色 + 眉型,即点即上。

## 3. 作用域规则(本次重构的核心)

### 3.1 Clear

- 点击 Clear:仅清除**当前一级页签品类**的妆(颜色置空 + 该品类手涂层清空 + 该品类色板选中态清除)。提示文案:"已卸除唇妆"/"已卸除腮红"等。
- **长按 Clear(550ms)**:全品类一键卸妆,提示"已全部卸妆(唇·颊·眉·眼)"。复用现有 attachLongPress。
- 衣橱页签下的 Clear 作用于唇妆(衣橱本质是唇妆的货架)。

### 3.2 按住对比

- 按住期间:仅当前品类不渲染,其余品类的妆**保留**——让用户看清单一产品的效果差异。
- 实现:`S.compare` 从 boolean 改为 `null | "lip" | "blush" | "brow" | "eye"`,drawMakeup 按值跳过对应品类。衣橱页签下对比作用于唇妆。

### 3.3 质地与浓度

- 均为品类私有状态;切页签时滑杆与质地 seg 恢复该品类记忆值(浓度记忆 v2.1 已实现,质地需从全局改为品类私有:lip.texture / blush.texture / eye.texture)。
- 眉妆页签:质地 seg 整体隐藏(display:none),底部行其余元素保持位置稳定。

## 4. 亲手涂抹扩展(腮红、眼影)

复用唇部已验证的"局部仿射空间涂抹层"方案(以面部锚点为基,涂层随头部运动无错位):

- **腮红涂抹空间**:以左右颧点(lm205/lm425)连线为基轴,独立 paint 画布(512²);锁区蒙版 = 两颊柔边大椭圆区域(同一键腮红的最大覆盖域),出界自动裁掉;渲染 = 涂抹层作为蒙版走 compositeMask 同款混合。
- **眼影涂抹空间**:以左右外眼角(lm33/lm263)连线为基轴;锁区蒙版 = 双眼上睑带(同一键眼影区域,上浮量放大 1.3 倍给涂抹余量)。
- 工具:大刷/小刷/卸妆棉,参数沿用唇部(刷径按区域尺度等比换算)。
- 每品类涂抹层独立持有,互不污染;一键与手涂在同品类内互斥覆盖(选一键色板即清该品类手涂层,反之进入手涂模式提示"在脸颊/眼皮上涂抹")。

## 5. 状态模型(目标形态)

```js
S.cat = "lip" | "blush" | "brow" | "eye" | "wardrobe"   // 一级
S.lip   = { mode:"oneclick|smear|shape", color, texture:"matte|gloss", int, paint, shapeParams }
S.blush = { mode:"oneclick|smear", color, style:"apple|sweep|sun", texture:"matte|shimmer", int, paint }
S.brow  = { color, shape:"natural|bold|arch", int }
S.eye   = { mode:"oneclick|smear", color, grad:false, texture:"matte|pearl", int, paint }
S.compare = null | "lip" | "blush" | "brow" | "eye"
```
v2.4 的全局 S.color/S.intensity/S.texture/S.maskMode 迁入 S.lip。允许 Code 按重构便利调整命名,但语义须等价。

## 6. 不变项(明确不许动)

渲染配方(color/multiply/深唇补偿/screen 高光)、肤色分析与推荐(★ 跨品类继续生效)、撞色预警、缩放/模糊背景、拍照、切后台自愈、隐私文案、版本号位置。

## 7. 验收标准

1. 五个一级页签可达,二级模式按矩阵齐全;眉妆页签无质地行、无手涂入口;
2. 在腮红页签 Clear:腮红消失,唇妆/眉妆/眼影原样保留;长按 Clear 全卸;四个品类逐一验证;
3. 在眼影页签按住对比:仅眼影消失,松手恢复;四品类逐一验证;
4. 腮红/眼影手涂:可涂、出界裁掉、卸妆棉可擦、转头跟随不错位;与一键互斥;
5. 微闪/珠光质地肉眼可辨(对照哑光);
6. 质地、浓度切页签往返记忆正确;
7. e2e:现有 44 项中作用域相关断言改写(Clear 全清 → 品类清 + 长按全清;对比 → 品类对比),新增手涂/质地断言;目标 ≥ 52 项,连续两轮全绿,部位锚定 + 色彩方向断言方法不变(__msDebug 钩子保留,补充颧点/眼角锚点输出);
8. 真机回归:唇妆全链路无退化。
