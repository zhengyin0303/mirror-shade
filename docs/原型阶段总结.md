# 镜色 MirrorShade · 原型阶段总结

> 状态:原型阶段收尾 | 最终版本:v1.7 | 日期:2026-06-11

## 一、阶段使命与结论

原型阶段的使命是验证两个问题:**技术上能不能做**(端侧实时唇部识别与自然渲染),**体验上好不好用**(真人真机的实际观感与手感)。

**结论:两者均已验证成立。**唇线识别在真机真人脸上贴合准确,涂抹/唇型编辑/质地渲染全链路可用,经多轮真机迭代后体验问题全部收敛。项目具备进入原生 App 阶段的条件。

## 二、最终功能清单(v1.7)

镜面预览(全屏镜像)· 开场肤色分析(冷暖调+深浅,确认收起/标签唤回/双维度手动修改)· 个性化推荐色板(★标记)· 一键试色(12色)· 亲手上妆(手指涂抹/锁区防出界/薄涂叠加/大小刷/卸妆棉)· 唇型编辑(8控制点拖动/微笑唇·嘟嘟唇·M唇预设/对称开关)· 哑光与水光质地 · 浓度滑杆 · **深唇显色补偿**(按唇色基底自适应混合)· 画面缩放(+/−按钮与双指捏合,缩小时模糊背景填充)· 按住对比素颜 · 拍照保存(系统分享面板)· Clear · ?帮助 · 切后台自愈 · 首屏隐私声明与版本号

文案已全球化:推荐理由表述为"更衬你的肤色"而非"显白",避免单一审美预设。

## 四、原型阶段沉淀的关键技术方案

1. **护牙蒙版**:唇外环+内环 evenodd 镂空,口腔区域天然排除;
2. **渲染配方**:color 混合(换色保唇纹)+ multiply(加深)+ 深唇遮盖通道(浓度平方加权),水光为蒙版内 screen 高光;
3. **涂抹坐标系**:以嘴角连线为基的唇部局部仿射空间,涂层随头部运动无错位(往返误差 10⁻¹⁴ 级);
4. **抗抖动**:位移自适应平滑(噪声抑制90%)+ 速度前瞻(补偿识别管线延迟);
5. **缩小观感**:全屏模糊背景填充(三稿迭代定稿);
6. **自动化测试**:Playwright + Mock 人脸数据源 + 假摄像头,26 项回归,无真实摄像头即可验证全部交互。

## 五、原生 App 需求清单(网页形态无法解决的)

1. **前置广角/物理镜头切换**——正常自拍构图的最终解(网页拿不到 Magic V3 的广角);
2. **拍照直存相册**——免确认写入(网页必须走分享面板);
3. 切后台体验、性能上限、推送、应用商店分发;
4. 多人多肤色的渲染参数标定(借助更大测试样本);
5. 口红衣橱+撞色预警、描唇练习打分、品牌色号库(二期功能,见项目方案第五节)。

## 六、下一步决策题(原生立项前需确定)

1. **平台**:iOS 先行 / Android 先行 / 双端(Flutter 或 React Native 跨端可显著降低双端成本,且本原型的渲染与算法逻辑可大量复用思路);
2. **开发资源**:自学+AI辅助 / 技术合伙人 / 外包(影响周期与成本量级);
3. **商业路径启动时点**:品牌色号库与导购合作在用户量达到什么规模时启动。
