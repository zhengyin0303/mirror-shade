# AR 口红试妆 App 项目方案 V1.0

> 状态:MVP 范围已确认 | 日期:2026-06-10

---

## 一、项目概述

做一款 AR 实时口红试妆应用。用户打开前置摄像头,手机即成为一面"会化妆的镜子":系统自动分析肤色并推荐显白色号,用户可一键试色或用手指亲手涂抹上妆,可调整唇型,不满意一键 Clear 重来。

**一句话定位:比专柜试色更准的诚实镜子 + 有化妆过程感的试妆体验。**

MVP 聚焦口红单一品类,把一个品类做到真实、好用、惊艳,后续再扩展品牌色号库与其他彩妆品类。

## 二、目标用户

核心用户为 18–35 岁女性,典型场景包括:网购口红前不确定颜色是否适合自己;不知道自己冷暖皮、想找显白色;化妆新手想练习画唇;以及单纯享受换妆乐趣的娱乐用户。

## 三、竞品分析与差异化定位

### 竞品现状(以玩美彩妆 YouCam Makeup 为代表)

竞品已非常成熟:2000+ 真实大牌色号、7 种唇膏质地、动态光影模拟、全品类彩妆、试妆直连购物车,且拥有十亿级下载量。**结论:在功能数量上正面竞争没有胜算,必须打结构性差异。**

### 竞品的结构性短板(我们的机会)

| # | 竞品短板 | 我们的打法 | 优先级 |
|---|---------|-----------|--------|
| 1 | 试妆与磨皮/瘦脸/美颜深度捆绑,试色结果失真,网购参考价值低 | **诚实试色**:默认零美颜,保留真实唇纹与肤色,定位"试色决定购买"的工具 | MVP |
| 2 | 全部是一秒上妆,没有"化妆的过程" | **亲手上妆**:手指涂抹、薄涂叠加、描唇练习,过程即乐趣 | MVP |
| 3 | 肤色匹配逻辑埋在色号标签里,没有做成核心体验 | **开场肤色分析**:打开即测冷暖皮,推荐显白色号,把线下数百元的"个人色彩诊断"做成免费开场仪式 | MVP |
| 4 | 商业模式靠品牌方付费,只帮品牌卖新品,不帮用户管理已有口红 | **口红衣橱 + 撞色预警**:录入已有口红,买前查重("与你已有的 XX 相似度 92%"),竞品在商业模式上难以跟进 | 二期 |

## 四、MVP 功能范围

### 4.1 核心流程

```
打开 App → 镜面预览 → 肤色分析(约3秒) → 推荐显白色板
   → 一键试色 / 亲手上妆(可先编辑唇型) → 长按对比素颜 → 拍照保存 / Clear 重来
```

### 4.2 功能清单(P0 = MVP 必须有)

| 模块 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| 镜面预览 | 前置摄像头实时画面 | 左右镜像,像照镜子 | P0 |
| 肤色分析 | 自动检测肤色深浅 + 冷暖调 | 脸颊/下颌/额头多区域采样、多帧取稳定值;引导自然光下检测;结果可手动修正(血管/首饰自测题兜底);话术用"为你推荐"不用"诊断" | P0 |
| 显白推荐 | 色板首排为个性化推荐 | 冷皮→蓝调红/玫红/浆果色,暖皮→橘调红/砖红/枫叶色 | P0 |
| 一键试色 | 点色即上唇 | 8–12 个通用经典色起步 | P0 |
| 亲手上妆 | 手指在屏幕上涂抹上色 | 颜色自动锁定唇区(涂出界自动裁掉);笔尖偏移光标;触唇时局部放大镜;大小两档笔刷;反复涂抹颜色逐层加深 | P0 |
| 唇型编辑 | 控制点调整唇型 | 自动生成贴合唇线的虚线,8 个可拖控制点(唇峰×2、唇谷、嘴角×2、下唇中央等);左右对称开关;预设:原生/微笑唇/嘟嘟唇/M唇 | P0 |
| 质地 | 哑光 / 水光 两种 | 哑光降反光、水光加高光点,均保留唇纹 | P0 |
| 浓度 | 薄涂→厚涂滑杆 | 与涂抹模式联动:涂得越多越浓 | P0 |
| Clear | 一键卸妆 | 唇型设置可选保留或一并重置 | P0 |
| 对比 | 长按屏幕显示素颜 | 松手恢复妆容,最低成本的前后对比 | P0 |
| 保存 | 拍照保存到相册 | 自带传播性 | P0 |
| 隐私声明 | 首屏明示"影像不离开手机" | 既是合规也是卖点 | P0 |

### 4.3 明确不做(写进范围纪律)

眼影、腮红、美瞳、粉底、发色、磨皮瘦脸等美颜功能、视频录制、社区——全部进二期清单,MVP 一律不做。**一个品类惊艳,胜过五个品类平庸。**

## 五、二期方向(按优先级)

1. **口红衣橱 + 撞色预警**:录入/拍照取色已有口红,试色时查重提示。需要一定用户基数后启动。
2. **描唇练习模式**:徒手沿引导线描唇,系统打分,带教学与游戏性。
3. **品牌色号库**:接入真实品牌/系列/色号,试妆→购买链接,开启导购佣金与品牌合作模式。
4. **左右半唇对比**:一张嘴同时试两个色号。
5. 视频录制换妆过程、分享模板;之后再考虑扩品类。

## 六、技术方案

**总体架构:全端侧处理,人脸画面不上传服务器。**

人脸识别采用 Google MediaPipe Face Mesh(开源免费,Web/iOS/Android 全支持),实时输出 468 个面部关键点,唇部内外轮廓有现成点位,同时支撑唇区蒙版、唇型控制点和肤色采样区域定位。

上色渲染不用纯色块覆盖,而是颜色混合模式(multiply / soft light)+ 边缘羽化,保留唇纹与光影;质地通过调节反光与高光实现。**唇部蒙版必须区分"嘴唇环"与"口腔内部",张嘴露齿时颜色不得染到牙齿**——这是本类产品最易翻车的点,列入验收标准。

肤色分析在 LAB 色彩空间计算深浅度与冷暖调,多帧平滑,异常光照时降级为引导用户手动选择。

**数据结构从第一天按"产品"建模**:每个颜色包含品牌、系列、色号、颜色值、质地、购买链接(暂空)字段,为二期品牌库与导购模式预埋,避免推翻重做。

### 开发路径

| 阶段 | 内容 | 目的 |
|------|------|------|
| 阶段 1 | 单文件网页原型(HTML + MediaPipe),手机浏览器直接打开 | 数天内验证上色自然度、跟脸延迟、张嘴/牙齿表现 |
| 阶段 2 | 原型上加入肤色分析、唇型控制点、质地浓度 | 完整走通 MVP 流程,真人多样本实测 |
| 阶段 3 | 原生 App 开发(平台待定:iOS / Android / 双端) | 上架版本,复用原型验证过的全部参数与逻辑 |

## 七、测试与验收要点

1. **张嘴露齿场景**:微笑、大笑、说话时颜色不染牙齿、不飘移。
2. **肤色与唇色多样性**:浅/中/深肤色、深唇色/浅唇色用户各一轮实测;渲染需考虑用户唇色基底(深唇薄涂偏暗,显色逻辑要符合真实口红表现)。
3. **光线鲁棒性**:自然光、暖黄室内光、冷白光下分别测试肤色分析准确率与上色观感。
4. **运动跟随**:转头、点头、说话时妆容贴合不抖动、不错位。
5. **涂抹体验**:手指粗细不影响结果(锁区生效),涂出界自动裁掉,叠涂加深曲线自然。

## 八、隐私与合规

人脸信息属最高敏感级个人信息(中国《个人信息保护法》、欧盟 GDPR 均严格监管)。方案:全程端侧运行,不采集、不上传、不存储人脸画面;首次使用明示并取得摄像头授权;肤色结果仅本地保存且话术为"推荐"而非"诊断"。隐私优势同时作为产品卖点在首屏呈现。

## 九、商业模式方向(远期)

MVP 阶段不商业化,专注体验。验证后的路径:① 品牌色号试妆→购买链接的导购佣金;② 品牌方虚拟试色合作/投放(Perfect Corp 已验证的模式);③ 口红衣橱衍生的个性化推荐。注意守住"诚实试色"定位,商业化不得引入失真美颜。

## 十、风险与应对

| 风险 | 应对 |
|------|------|
| 上色不够自然,用户觉得"假" | 原型阶段死磕混合模式与质地参数,以真人盲测"像不像真涂了"为标准 |
| 环境光导致肤色误判 | 自然光引导 + 多帧平滑 + 用户手动修正兜底 + 推荐式话术 |
| 竞品体量碾压 | 不比功能数量,坚守四块差异化:诚实试色、过程体验、肤色开场、口红衣橱 |
| 范围蔓延 | 二期清单制度,MVP 只做第四节 P0 功能 |
